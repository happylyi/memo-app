# 个人备忘录 App V1.3 产品需求文档

## 1. 产品概述

个人备忘录 App 是一款轻量级纯前端备忘录工具，用于日常记录待办事项、灵感想法等简短内容，无需后端服务，开箱即用。

---

## 2. 核心需求

- 支持新增备忘录，输入内容限制50字以内，保存后显示在列表顶部
- 支持编辑备忘录，点击编辑按钮后内容回填到输入框，保存后更新原条目
- 支持删除备忘录，点击删除按钮后弹出确认弹窗，确认后移除
- 支持标记重要备忘录，点击标题切换红色加粗样式，再次点击恢复默认
- 支持数据导出，点击导出按钮将所有备忘录下载为 JSON 文件

---

## 3. 功能详情

### 3.1 新增备忘录

| 项目 | 描述 |
|------|------|
| 入口 | 页面顶部输入区域 |
| 输入控件 | textarea 多行文本输入框，maxlength=50，支持换行 |
| 字数提示 | 输入框下方实时显示"已输入字数/50" |
| 保存按钮 | 点击「保存」按钮提交内容 |
| 校验规则 | 内容为空时提示"请输入备忘录内容" |
| 保存成功 | 清空输入框，新备忘录插入列表顶部，显示创建时间 |
| 数据存储 | 自动保存至 localStorage |

### 3.2 编辑备忘录

| 项目 | 描述 |
|------|------|
| 入口 | 每条备忘录右侧的「编辑」按钮（蓝色 #2196F3） |
| 编辑模式 | 点击编辑后，按钮变为「更新」，输入框回填原内容 |
| 保存更新 | 点击「更新」按钮保存修改，只更新当前条目，不新增 |
| 保存成功 | 清空输入框，恢复为新增模式，按钮变回「保存」 |
| 数据存储 | 更新后自动同步到 localStorage，刷新不丢失 |
| 状态保留 | 编辑时保留原备忘录的重要标记状态 |
| 边界处理 | 删除正在编辑的备忘录时，自动清除编辑状态 |

### 3.3 删除备忘录

| 项目 | 描述 |
|------|------|
| 入口 | 每条备忘录右侧的「删除」按钮 |
| 确认机制 | 点击删除按钮后弹出确认弹窗，显示"确定要删除这条备忘录吗？" |
| 弹窗操作 | 点击「确定」执行删除，点击「取消」或遮罩层关闭弹窗 |
| 删除成功 | 从列表中移除该条目，同步更新 localStorage |
| 空状态 | 列表为空时显示"暂无备忘录，快来添加吧！"提示 |

### 3.4 标记重要

| 项目 | 描述 |
|------|------|
| 入口 | 点击备忘录标题区域（非按钮区域） |
| 样式切换 | 默认样式 ⇄ 红色加粗样式 |
| 状态记录 | 重要状态保存至 localStorage，刷新页面保持 |
| 视觉反馈 | 切换时标题文字变为红色并加粗，再次点击恢复黑色常规字重 |

### 3.5 数据完整性保障（V1.2.0 新增）

| 项目 | 描述 |
|------|------|
| 用户场景 | 用户希望刷新页面后数据不丢失、不乱序 |
| 实时同步 | 每次新增/编辑/删除/标记操作后立即同步 localStorage |
| 数据读取 | 页面加载时完整读取存储数据，保证顺序与内容一致 |
| 数据完整性 | 无数据丢失、无格式错乱 |
| 版本管理 | 存储版本号，支持旧版本数据自动迁移 |
| 错误处理 | 存储失败时提示用户，读取失败时使用空数组初始化 |

### 3.6 数据导出（V1.2.0 新增）

| 项目 | 描述 |
|------|------|
| 用户场景 | 用户希望备份备忘录数据 |
| 入口 | 页面底部绿色「导出备忘录 (JSON)」按钮 |
| 导出格式 | JSON 文件，包含版本号、导出时间、总数、所有备忘录详情 |
| 文件命名 | 格式为「备忘录_日期.json」，如：备忘录_2026-03-16.json |
| 用途 | 文件可用于备份或迁移到其他设备 |
| 移动端适配 | 导出按钮响应式布局，移动端自动调整宽度 |

---

## 4. 非功能需求

| 类别 | 要求 |
|------|------|
| 界面设计 | 采用卡片式布局，配色简洁清爽，操作直观易用 |
| 移动端适配 | 响应式设计，支持 320px ~ 1200px 屏幕宽度，移动端按钮组自动换行 |
| 数据存储 | 使用浏览器 localStorage 持久化存储，无需后端服务，刷新页面数据不丢失 |
| 兼容性 | 支持主流现代浏览器（Chrome、Firefox、Safari、Edge） |
| 性能要求 | 纯前端实现，无网络请求，页面加载快速，操作响应流畅 |
